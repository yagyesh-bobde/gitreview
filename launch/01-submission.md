## Maker's first comment (post immediately after going live)

> 70% of Product-of-the-Day winners had a maker first comment. Post this within the first minute. Tone: a developer talking to other developers, not a press release. **Write/finalize it 48h early and read it aloud.**

```
Hey Product Hunt! 👋 I'm Yagyesh, the maker of GitReview.

This started with a PR I'll never forget — a 600-file refactor I was asked
to review. GitHub's diff view choked, I lost track of which files actually
mattered, and I basically rubber-stamped it because genuinely reviewing it
was impossible. That's the worst feeling as an engineer: approving code you
didn't really read.

Every "AI code reviewer" I tried was built for small, tidy PRs. None of them
helped with the 500-file monsters that actually need help. So I built
GitReview for exactly that:

🗂️ Intelligent file grouping — see the shape of a PR instead of an
   alphabetical wall of files
⚡ Virtualized diff viewer — scroll 1,000+ files without the browser dying
🤖 Gemini AI assistant — ask "what does this change do?" or have it flag
   risky diffs, on every screen
⌨️ Keyboard-driven — navigate, comment, and approve without the mouse
🔄 Two-way GitHub sync — your comments and approvals land right back on the PR

It's free, and free for open source.

🎁 Product Hunt fast-track: comment with how you handle large PRs today
(split them? skim them? avoid them?) and I'll give you priority early access
to what we're building next — plus a "founding reviewer" badge.

I'll be here all day answering everything. What would make GitReview a
no-brainer for your team? 🙏
```

**Notes:**
- This asks for **comments and feedback**, never upvotes — fully compliant.
- The "fast-track" offer doubles as a comment-driver (cubic won #1 on comment volume). Swap in a real perk if you have one (e.g. lifetime Pro for the first N commenters).
- If GitReview ships a paid tier before launch, you can additionally offer a code like `PRODUCTHUNT` (Trag won #1 with `TRAGPH` 50%-off-3-months in its first comment).

## Pre-filled reply snippets (for fast, on-brand responses)

Keep these in a notes app on launch day so you can reply within ~10 min to every comment.

**To "how is this different from CodeRabbit / cubic / etc.?"**
```
Great question. Most AI reviewers post inline comments on normal-sized PRs.
GitReview is the actual review *surface* for when a PR is too big to read in
GitHub at all — smart file grouping, a virtualized diff that won't choke on
1,000 files, and AI you can ask about any change. It's the cockpit, not just
the bot.
```

**To "does it work with GitLab / Bitbucket?"**
```
GitHub today (with two-way sync so your work stays where your team expects
it). [GitLab/Bitbucket] is on the roadmap — want me to ping you when it
lands?
```

**To a feature request:**
```
Love this — added to the list. If you drop your use case I'll fast-track you
early access when it ships. What's the largest PR you've had to review lately?
```

**To praise:**
```
Thank you 🙏 means a lot. If you end up reviewing a big PR with it, I'd
genuinely love to hear what worked and what didn't.
```
